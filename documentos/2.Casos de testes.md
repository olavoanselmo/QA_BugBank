# 1. Introdução  

## 1.1 Identificador do Documento:  

**Nome Projeto:** Bug bank  <br>
**Versão Atual**: 0.2  <br>
**Documento:** Aprovado e emitido em 30 de agosto de 2026. <br>
**Equipe Responsável:**  Olavo Cavalcanti Anselmo  Engenheiro de Qualidade  de Software.
  
## 1.2 Escopo:  

O BugBank  é um sistema que simula um app para banco, open source,  que oferece ao usuário algumas funcionalidades comuns em um site de banco. O sistema possibilita realizar navegação pela home do site e simula algumas funcionalidades próprias de um app de banco, tais como realizar o cadastro de conta, efetuar  login, transferências e extratos, as funcionalidades de saque e pagamento ainda não estão disponíveis, estavam disponíveis no momento dos testes.

## 1.3 Referências Internas:  

**Documento 1:** Plano de Testes (Disponível em: 24/08/2026)  

## 1.4 Notação para Especificações dos casos de testes:  

- Para representar o ID dos cenários de teste iremos usar a sigla CN (cenário de testes) apenas a numeração simples. Ex: CN01: Cadastro 
- Para representar o ID dos casos de teste iremos usar a sigla CT (Caso de teste) acompanhado de seu ID. Ex: CT001 : Cadastro sem os campos obrigatórios.

Template que será seguido:  


Cenário XX |  Nome do Cenário | 
|:---|:---|
ID | CTXXX
Autor(a) | Nome do profissional responsável pela criação do cenário. 
Nome do CT | Nome do caso de teste que será especificado. |
Prioridade | Prioridade com que o caso de teste deve ser tratado. |
Ambiente | Local e estrutura onde os testes serão realizados. |
Pré-condições | O que deve é necessário para que a tarefa seja realizada. |
Dados de Entrada | Dados que o testador deve fornecer ao sistema. |
Passo a Passo | Passo a passo que o testador deve seguir. |
Resultado | Esperado O que é esperado que o sistema retorne após a ação. |
Resultado Atual | Resultado que o sistema exibiu após os testes. |
| Observação  | Observação que o testador pode fazer. |
